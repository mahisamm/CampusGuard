# Environment Variables Reference

Complete documentation of all environment variables used in CampusGuard.

## Variable Format & Loading

### Development

Variables can be set:
1. Shell export: `export KEY=value`
2. `.env` file (not committed): `KEY=value`
3. Docker Compose: defined in `docker-compose.yml`
4. Command-line: `KEY=value java -jar app.jar`

### Production (Render/Vercel)

Variables set via platform dashboards:
1. **Render**: Dashboard → Service → Settings → Environment Variables
2. **Vercel**: Dashboard → Settings → Environment Variables

### Database Configuration

| Variable | Default | Required | Purpose |
|----------|---------|----------|---------|
| `DATABASE_URL` | `jdbc:h2:mem:college_connect;...` | ✗ (local dev) | JDBC connection string |
| `DATABASE_USERNAME` | `sa` | ✗ | Database user |
| `DATABASE_PASSWORD` | `` | ✗ | Database password |
| `DATABASE_DRIVER` | `org.h2.Driver` | ✗ | JDBC driver class |
| `HIBERNATE_DIALECT` | `org.hibernate.dialect.H2Dialect` | ✗ | ORM SQL dialect |

**Local Development (H2 in-memory, default):**
```bash
export DATABASE_URL=jdbc:h2:mem:college_connect;DB_CLOSE_DELAY=-1;MODE=PostgreSQL
export DATABASE_DRIVER=org.h2.Driver
export HIBERNATE_DIALECT=org.hibernate.dialect.H2Dialect
```

**Local Development (PostgreSQL):**
```bash
export DATABASE_URL=jdbc:postgresql://localhost:5432/college_connect
export DATABASE_USERNAME=college_admin
export DATABASE_PASSWORD=college_password
export DATABASE_DRIVER=org.postgresql.Driver
export HIBERNATE_DIALECT=org.hibernate.dialect.PostgreSQLDialect
```

**Production (Render, auto-injected):**
```
DATABASE_URL: postgresql://college_admin:xxxxx@aws-xxxxx.c179.us-east-1.pg.rds.amazonaws.com:5432/college_connect
DATABASE_USERNAME: college_admin
DATABASE_PASSWORD: (auto-generated by Render)
DATABASE_DRIVER: org.postgresql.Driver
HIBERNATE_DIALECT: org.hibernate.dialect.PostgreSQLDialect
```

### Security & JWT

| Variable | Default | Required | Purpose |
|----------|---------|----------|---------|
| `JWT_SECRET` | (dev hardcoded) | ✗ (local) | Token signing key **(min 32 chars)** |
| `JWT_EXPIRATION` | `86400000` | ✗ | Token TTL in milliseconds |

**Local Development:**
```bash
# Use default (NOT secure for production)
JWT_SECRET=404E635266556A586E3272357538782F413F4428472B4B6250645367566B5970
JWT_EXPIRATION=86400000  # 1 day
```

**Production (Render):**
```bash
# Render auto-generates via: generateValue: true in render.yaml
# Or manually set to secure random (32+ chars):
JWT_SECRET=$(openssl rand -hex 32)
# Output example: 5d41402abc4b2a76b9719d911017c592
```

**Token Expiry Values:**
```
1 day = 86400000
7 days = 604800000
30 days = 2592000000
```

### CORS Configuration

| Variable | Default | Required | Purpose |
|----------|---------|----------|---------|
| `ALLOWED_ORIGINS` | `http://localhost:5173,http://localhost:3000` | ✗ | Comma-separated CORS whitelist |

**Local Development:**
```bash
ALLOWED_ORIGINS=http://localhost:5173,http://localhost:3000
```

**Production (Render, open for frontend):**
```bash
ALLOWED_ORIGINS=*
```

**Production (Restricted):**
```bash
ALLOWED_ORIGINS=https://app.yourdomain.com,https://www.yourdomain.com
```

## Frontend Environment Variables

### API Configuration

| Variable | Default | Required | Purpose |
|----------|---------|----------|---------|
| `VITE_API_URL` | `` (empty) | ✗ | Backend API base URL |

**Local Development (Vite proxy, default):**
```bash
# Leave empty, Vite proxies /api to localhost:8080
VITE_API_URL=
```

**Local Production Test:**
```bash
VITE_API_URL=http://localhost:8080
```

**Production (Vercel, auto-injected by render.yaml):**
```bash
VITE_API_URL=https://college-connect-api-xxxx.onrender.com
```

## Docker Compose Environment Variables

Variables can be set in `docker/.env` (not committed):

```env
JWT_SECRET=dev_jwt_secret_key_xxxxxxxxxxxxxxxxxxxxxxxx
POSTGRES_PASSWORD=college_password
PORT=8080
```

**Load from file:**
```bash
cd docker
docker-compose --env-file .env up
```

Or inline:
```bash
JWT_SECRET=my_secret PORT=9000 docker-compose up
```

## Secrets Management Best Practices

### Do NOT Commit Secrets

- ✗ Hardcode password in code
- ✗ Add `.env` file to git
- ✗ Commit API keys

### DO Manage Secrets Safely

- ✓ Use `.env` locally (in `.gitignore`)
- ✓ Use `.env.example` as template in git
- ✓ Store secrets in platform dashboards (Render, Vercel)
- ✓ Rotate secrets regularly
- ✓ Use unique secrets per environment

### Audit for Secrets

```bash
# Search git history for secrets
git log -S "password" --all   # Password
git log -S "secret" --all     # Secrets
git log -S "key" --all        # Keys

# Remove from history (if found)
# Use git-filter-repo or BFG Repo-Cleaner
```

## Troubleshooting

### "Unknown property 'jwt.secret'"

**Cause**: Env var not loaded

**Solution**: Ensure `JWT_SECRET` is set before running backend

```bash
export JWT_SECRET=my_secret
java -jar target/*.jar
```

### "JDBC Connection failed: Cannot get a connection"

**Cause**: DATABASE_URL is invalid

**Solution**: Verify connection string format

```
H2:         jdbc:h2:mem:college_connect
PostgreSQL: jdbc:postgresql://host:5432/dbname
```

### "CORS header 'Access-Control-Allow-Origin' missing"

**Cause**: Frontend origin not in `ALLOWED_ORIGINS`

**Solution**: Update and redeploy backend

```bash
export ALLOWED_ORIGINS=http://localhost:5173
java -jar target/*.jar
```

### "API calls fail with 401 Unauthorized"

**Cause**: JWT_SECRET changed after token issued

**Solution**: Clear localStorage and re-login

```javascript
// Browser console
localStorage.removeItem("jwt_token");
// Refresh page and log in again
```
